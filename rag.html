🧠 Key Concepts of RAG

  • Chunking: Breaks large documents into smaller semantic pieces.
  • Overlap: Preserves context between chunks.
  • Embeddings: Numerical semantic representation of text.
  • Vector Database: Stores embeddings for fast similarity search.
  • Semantic Search: Finds meaning-based matches, not keyword matches.
  • Retrieval: Fetches the most relevant chunks.
  • LLM Generation: The LLM uses retrieved context to answer accurately.
  • Important: The query itself is also converted into an embedding.
